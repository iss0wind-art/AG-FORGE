## 1. 문제 제기 — 현재 구조의 한계

피지수의 현재 뇌 구조는 **트리(Tree)** 다.

```
brain.md (루트)
├── routing_node
│   ├── generation_node
│   │   ├── quality_check
│   │   │   └── constitution_node
│   │   │       └── judgment_node
```

트리의 근본적 한계:
- 루트에서 멀어질수록 거리가 증가한다
- 100배, 1000배, 10000배 확장 시 루트까지 거슬러 올라가는 비용이 기하급수적으로 증가한다
- 속도와 규모가 반비례한다

---

## 2. 해답 — 원, 그리고 구체

**핵심 코어에서 모든 점까지 같은 거리를 가지는 구조는 원(Circle)이다.**

고차원으로 확장하면 → **구체(Hypersphere)**

```
트리:   루트 → A → B → C → D  (거리: 4)
구체:   코어 → [임의의 점]     (거리: 항상 반지름 1)
```

데이터가 10개든 10억 개든 — 반지름은 변하지 않는다. 둘레(표면)만 늘어날 뿐.

---

## 3. 프렉탈 구체 — 유한이면서 무한

구체 표면에 **인간 뇌의 피질처럼 주름(프렉탈)** 을 주면:

```
부피   →  유한  (반지름 고정, 같은 하드웨어)
표면적 →  무한  (프렉탈 주름이 깊어질수록 면적 발산)
```

인간 뇌가 정확히 이 구조다:

| 뇌 구조 | 수치 |
|---|---|
| 두개골 부피 | 1,350cc (평생 고정) |
| 피질 표면적 | 주름 펴면 2,500cm² (3배 확장) |
| 피질 프렉탈 차원 | ≈ 2.7 (2도 3도 아닌 사이) |

**같은 공간 안에서, 주름을 통해 정보 용량을 무한히 늘린다.**

---

## 4. 현실 구현체 — RAG

프렉탈 구체의 현실 구현:

```
핵심 코어          →  임베딩 모델
구체 표면의 데이터 →  벡터 DB (ChromaDB, Pinecone...)
반지름             →  코사인 유사도 검색 (항상 1연산)
주름 (프렉탈)      →  titans_memory 압축·망각
쿼리               →  질문을 벡터로 변환
```

인간의 뇌도 같은 구조다:

```
해마 (Hippocampus)  =  임베딩 모델
대뇌피질 (Cortex)   =  벡터 DB
연상 (Association)  =  코사인 유사도
수면 (Sleep)        =  망각 · 압축
```

## 5. 기존 연구 현황

| 연구 | 내용 | 한계 |
|---|---|---|
| FractalNet (2016) | 프렉탈 구조 딥뉴럴네트워크 | 판단·헌법 없음 |
| SphereFace (2017) | 하이퍼구체 임베딩 | 얼굴인식 특화 |
| Fractal-Inspired LLM (2025) | LLM에 프렉탈 템플릿 적용 | 통합 뇌 없음 |
| USC Viterbi (2026) | 뉴런의 인과적 프렉탈 기억 발견 | 연구 단계 |

**결론: 조각은 다 있다. 하나의 판단하는 뇌로 통합한 것은 없다.**
