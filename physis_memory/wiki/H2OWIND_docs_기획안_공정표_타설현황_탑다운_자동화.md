## 1. 배경 및 목표

### 현재 방식 (Bottom-up)
- 오늘 날짜 기준으로 앞으로 공정을 입력
- 공정표를 직접 수기 입력 → 시간 소모 + 오류 가능성

### 새로운 방식 (Top-down 추가)
- **최상층 타설일 입력 → 아래 층수 자동 역산**
- 공정 사이클(타설 주기)을 기준으로 전 층 날짜 자동 생성
- 오늘 이전: 실제 확정 공정 (수정 불가 구간)
- 오늘 이후: 계획 공정 (미확정 = 회색, 확정 = 해당 월 색상)

### 핵심 원칙
```
최상층 타설일 → 역산 → 전 층 날짜 자동 채움
                       ↓
              오늘 이전: 실제 데이터 (불변)
              오늘 이후: 계획 (회색 = 미확정)
                         관리자 확정 시 → 월 색상 부여
```

---

## 2. 시스템 구조

### 2-1. 현장 기본 설정 (프로젝트 셋업)
공정표와 타설현황이 공유하는 마스터 설정

```
현장 기본 설정
├── 현장명 (데이터 저장소 식별)
├── 회사명
├── 동 구성
│   ├── 동 시작 번호 (예: 101)
│   ├── 총 동수 (예: 16)
│   └── 동별 층수 (B2~16F 등 개별 설정 가능)
├── 층 구성
│   ├── 지하 시작 (예: B2 = -2)
│   ├── 총 지상 층수
│   └── 층별 높이/슬래브 두께
└── 공정 사이클
    ├── 기본 타설 주기 (예: 7일)
    └── 동별/층별 예외 주기 설정
```

**저장 위치**: `SystemConfig` 테이블 (key-value) 또는 신규 `ProjectSetup` 테이블

---

### 2-2. 탑다운 자동화 플로우

```
[입력]
최상층 타설일 입력 (동별)
예) 101동 16F → 2026-08-15

[처리]
공정 사이클(7일 기준) × 층수 역산
16F: 2026-08-15
15F: 2026-08-08
14F: 2026-08-01
...
B1F: 2026-03-12  ← 오늘(2026-04-14) 이전 = 실제 데이터
...

[출력 규칙]
- 오늘(2026-04-14) 이전 날짜 → 건드리지 않음 (이미 실적 데이터)
- 오늘 이후 날짜 → 회색(#CCCCCC) = "계획(미확정)" 상태로 저장
- 관리자가 개별 확정 시 → 해당 월 색상으로 변경
```

---

### 2-3. 색상 체계 (월별 색상 + 상태)

| 상태 | 색상 | 설명 |
|------|------|------|
| 계획(미확정) | 회색 `#CCCCCC` | 탑다운 자동 생성, 아직 미확정 |
| 1월 확정 | 파랑 계열 | 관리자 확정 처리 |
| 2월 확정 | 초록 계열 | 관리자 확정 처리 |
| 3월 확정 | 노랑 계열 | 관리자 확정 처리 |
| ... | ... | 기존 타설현황 색상 체계 유지 |
| 과거 실적 | 원래 색상 | 오늘 이전 = 수정 불가 |

---

### 2-4. 공정표 ↔ 타설현황 동기화

```
[데이터 모델]
타설 날짜 1건 = {
  동: "101동",
  층: "16F",
  날짜: "2026-08-15",
  상태: "confirmed" | "planned",  // confirmed=확정, planned=미확정
  색상: "월별색상" | "gray",
  수정여부: boolean,              // 관리자 수동 수정 여부
  출처: "topdown-auto" | "manual" // 자동생성 vs 수동입력
}

공정표(Gantt) ──┐
                ├── 동일 데이터 참조 (양방향 동기화)
타설현황         ──┘
```

**동기화 규칙**:
- 타설현황에서 날짜 변경 → 공정표 자동 반영
- 공정표에서 날짜 변경 → 타설현황 자동 반영
- 단, "수동 수정 플래그"가 있는 셀은 탑다운 재계산 시 덮어쓰지 않음

## 3. 단계별 구현 계획

### Phase 1: 현장 설정 통합
- [ ] `ProjectSetup` 모델 설계 (동수/층수/공정사이클)
- [ ] 기존 설정창 UI를 마스터 설정으로 통합
- [ ] 공정표 + 타설현황이 동일 설정 참조하도록 연결

### Phase 2: 탑다운 자동화 엔진
- [ ] 최상층 날짜 → 전 층 역산 함수 구현
- [ ] 오늘 이전 날짜 보호 로직 (실적 데이터 불변)
- [ ] 회색(미확정) 상태로 저장 로직
- [ ] 동별 예외 사이클 처리

### Phase 3: 확정 처리 UI
- [ ] 미확정(회색) 셀 클릭 → 확정 처리 버튼
- [ ] 확정 시 해당 월 색상 자동 부여
- [ ] 확정 취소 → 다시 회색으로

### Phase 4: 공정표 자동화 연동
- [ ] 타설현황 날짜 → 공정표 바(Bar) 자동 생성
- [ ] 양방향 동기화 구현
- [ ] 변경 이력 저장 (AuditLog 활용)

## 4. DB 변경 사항

### 신규 테이블: `ProjectSetup`
```sql
CREATE TABLE ProjectSetup (
  id          TEXT PRIMARY KEY,
  siteId      TEXT NOT NULL,       -- 현장 식별자
  siteName    TEXT NOT NULL,       -- 현장명
  companyName TEXT,
  buildingStart INTEGER,           -- 동 시작 (101)
  buildingCount INTEGER,           -- 총 동수 (16)
  floorMin    INTEGER,             -- 지하 최저층 (-2 = B2)
  floorMax    INTEGER,             -- 최고 층수 (16)
  defaultCycle INTEGER DEFAULT 7,  -- 기본 타설 주기(일)
  createdAt   TEXT,
  updatedAt   TEXT
)
```

### 기존 테이블 수정: `DashboardStore` (간트차트 데이터)
```
status     TEXT  -- "confirmed" | "planned"
source     TEXT  -- "topdown-auto" | "manual"
isLocked   INTEGER DEFAULT 0  -- 수동 수정 보호 플래그
```

## 5. UI/UX 변경 사항

### 타설현황 (Concrete Matrix)
- 상단에 "탑다운 자동 생성" 버튼 추가
- 최상층 날짜 입력 모달 (동별)
- 회색 셀 = 미확정 계획, 클릭 시 확정 처리 가능
- 확정된 셀 = 월 색상, 우클릭 → 날짜 수정 (2차 확정)

### 공정표 (Gantt)
- 탑다운으로 생성된 바(Bar)는 점선 테두리 = 미확정
- 확정된 바 = 실선 + 월 색상
- 타설현황과 실시간 동기화 표시

## 6. 미결 사항 (방부장 확인 필요)

1. **공정 사이클**: 동별로 다른가, 아니면 전 현장 동일한가?
2. **탑다운 재계산**: 중간 층 날짜를 수동 수정 후 → 위/아래 재계산 범위?
3. **과거 실적 연동**: 타설현황의 기존 B1/B2 데이터가 이미 있음 → 마이그레이션 방식?
4. **BOQ 연동 시점**: 현재는 팝아이즈 단독, BOQ 연동은 다음 단계?
5. **다중 현장 지원**: 현장 전환 기능은 이미 있음 → 각 현장마다 별도 공정 사이클?
