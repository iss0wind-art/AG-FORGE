## A. 분할 전략 — 레이어 분할 + 청크 보강 하이브리드

**채택안**: (ii) 레이어 분할 기본 + (iii) 청크 보조

```
1단계: ezdxf.readfile() → entity.dxf.layer 기준 그룹화
  └─ 건설 도메인 5그룹:
       G1. 기둥 레이어  (COL, COLUMN, C-COL 등)
       G2. 보 레이어    (BEAM, C-BM, S-BM 등)
       G3. 슬라브 레이어 (SLAB, C-SL 등)
       G4. 벽체 레이어  (WALL, C-WL, SW 등)
       G5. 기타 레이어  (DIM, TEXT, ANNO, GRID 등)

2단계: 각 그룹 entity 수 > 3,000 시 → 좌표 기반 청크 분할
  └─ BBox 계산 후 X축 또는 Y축 등분 (의미 경계 근사)
```

**why 레이어 우선**: pattern_library §B-1 "XR 레이어 식별" 이미 정합.
레이어명이 부재 의미를 담고 있어 LLM 정확도 최대화.

## B. 5체 정반합 — LLM 역할 분담

| 체 | LLM | 담당 레이어 | 강점 |
|----|-----|-------------|------|
| 1체 | DeepSeek R1 | G1 기둥 | 수치·치수 정밀도 1위 |
| 2체 | Gemini 1.5 Pro | G2+G3 보+슬라브 | 긴 컨텍스트, 면적 계산 |
| 3체 | Groq Llama | G4 벽체 | 속도 최고, 단순 패턴 |
| 4체 | Claude Sonnet | G5 기타 | 의미 이해, 이상치 해석 |
| 5체 (합) | DeepSeek R1 | 전체 통합 | 4체 결과 정반합 |

```python
async def five_phase_synthesis(layer_groups: dict) -> dict:
    tasks = [
        call_deepseek(layer_groups['기둥']),
        call_gemini(layer_groups['보'] + layer_groups['슬라브']),
        call_groq(layer_groups['벽체']),
        call_claude(layer_groups['기타']),
    ]
    results = await asyncio.gather(*tasks)
    return await call_deepseek(synthesis_prompt(results))  # 5체 합
```

## E. pattern_library 갱신 자동화 — YES

```
3/5 이상 LLM 동의 → 패턴 확정 → pattern_library.md 자동 append
                                → ChromaVectorIndex 임베딩 저장
                                → physis_memory/wiki/log.md 이력 기록
```

갱신 거버넌스:
- 새 패턴 신뢰도 = (동의 LLM 수 / 5) × max_surprise_score
- 신뢰도 < 0.4 → "후보" 상태 (이천 수동 확인 후 확정)
- 신뢰도 ≥ 0.4 → 자동 확정
