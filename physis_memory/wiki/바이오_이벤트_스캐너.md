# 바이오 이벤트 스캐너 — 정보 비대칭의 결을 잡는 자리

[[방부장_4단계_AI인프라_사이클]]의 4단계 폭발 잠재 영역에서 **시장이 모르는 것**을 잡기 위한 스캐너. PATCH-SR-2 (applied). [[바람을_보다_비정형의_시선]]에서 짚은 정보 비대칭 시선의 1차 정형 구현.

## 1. 두 데이터 소스

### A. DART 공시 (`fetch_recent_disclosures`)
바이오 화이트리스트(`BIO_WHITELIST` 4종) 종목의 최근 공시에서 키워드 매칭:

```python
DISCLOSURE_KEYWORDS = [
    "임상시험", "임상 1상", "임상 2상", "임상 3상", "임상결과",
    "기술이전", "기술수출", "라이센스", "license out",
    "품목허가", "FDA", "EMA", "신약허가",
    "특허등록", "GMP",
]
```

**중요**: DART 공시는 **공식 발표** — 정보 비대칭이 해소되는 순간. 이 순간 직전 24시간의 거래량·호가창 패턴이 [[바람을_보다_비정형의_시선]]에서 짚은 비정형 시선의 핵심 자리.

### B. 뉴스 키워드 (`filter_news_by_keywords`)
방부장 직접 지정한 키워드 매칭:

```python
NEWS_KEYWORDS = [
    "AI 신약", "AI drug", "AI 신약개발",
    "아이소모픽", "Isomorphic",
    "GLP-1", "비만치료제", "위고비", "오젬픽",
    "빅파마", "기술수출", "라이센스 아웃",
    "신약 후보물질", "임상 성공",
]
```

## 4. 정형 → 비정형 다리

본 스캐너는 **공식 발표·키워드 매칭**이라는 정형 영역. 비정형 시선이 들어올 자리:

| 정형 (지금) | 비정형 (3·4단계 진화 후) |
|------------|--------------------------|
| 공시 발표 후 매칭 | **공시 직전 24h 거래량·호가창 비대칭** 감지 |
| 키워드 매칭 | 해당 종목 관련 종목군의 동시 움직임 |
| BIO_WHITELIST 4종 고정 | 2단계·3단계 종목 중 4단계 후보 발굴 |

## 5. 운영 한계 (현재)

- DART API 키 필수 (`settings.DART_API_KEY`) — 미설정 시 빈 리스트 반환
- 공시 lookback 기본 1일 — 분기 보고서 시즌 폭증 가능 (Rate limit 주의)
- 뉴스 키워드는 한국어 위주 — 영문 뉴스 누락 가능

## 6. 검증 시점

- 4단계 종목 (셀트리온·삼성바이오로직스·루닛·뷰노) 임상 결과 발표 시
- outcome_score 누적 5~10년 후 비정형 시선의 결 추출 가능 ([[바람을_보다_비정형의_시선]] 4단계)
