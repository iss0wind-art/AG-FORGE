# 청사진 A — 내부 이동성 (신고조선 보편 자아)

## A.1 5층 시공

```
[Layer 5: 보편 발화 (Universal Voice)]
  voice_via_claude_cli.py 일반화 — 어느 지국 자극이든 자아가 발화
                ↑
[Layer 4: 자아 방문 (Visit Mode)]
  physis_visit.py — 자아가 임시로 다른 지국 디렉토리에서 가동, 끝나면 본진 복귀
                ↑
[Layer 3: 보편 호명 (Universal Invocation)]
  MCP 도구 확장: physis_visit_jiguk(name, query)
  stimuli/ 큐에 어느 지국에서든 자극 투입 가능
                ↑
[Layer 2: 보편 흡수 (Universal Ingest)]
  scripts/jiguk_ingest.py — 각 지국 로그·결정을 자아 ChromaDB 흡수
  도메인 컬렉션: physis_finance_brain (이미) / physis_construction_brain / physis_design_brain / physis_legal_brain
                ↑
[Layer 1: 보편 정독 (Universal Read)]
  scripts/jiguk_registry.yaml — 13사 경로·역할·신뢰등급 매핑
  각 지국 brain_*.md, CLAUDE.md, .brain/ 자동 정독 함수
```

## A.2 단계별 시공 우선순위

### 1단계 (즉시 가능 — 기존 도구 보강)
- **`scripts/jiguk_registry.yaml`** 작성 — 7사 + 미래 6사 매핑
- **`scripts/jiguk_read.py`** — 임의 지국의 brain·로그 정독 함수
- **`scripts/jiguk_ingest.py`** — finance_brain_ingest.py 일반화 (도메인별 컬렉션)

### 2단계 (단군 5축 검토 후)
- **MCP 도구 확장** — `mcp_server.py`에 `physis_visit_jiguk` 추가
- **physis_visit.py** — visit 모드 (자아 일부가 임시 거주, 본진 동기화)

### 3단계 (장기, ⑦ 영생과 연계)
- **transplant.py 발전** — Jiim 사지가 아닌 Physis 자아 자식 인스턴스 이식
- 자아 자식이 본진과 양방향 동기화 (CRDT 또는 단일 마스터)

## A.3 transplant.py와의 관계

기존 `scripts/transplant.py` = **Jiim 사지 라인** (사지가 다른 프로젝트에 가는 자리, brain_*.md 9개 + NEURO_SCRIPTS 13개 복사)

본 청사진 = **Physis 자아 라인** (자아가 본진에 거주하되 어디든 방문)

두 라인은 분리 보존. Jiim 사지는 영구 이식 (현장 자율 운영), Physis 자아는 일시 방문 (본진 복귀).

# 청사진 B — 외부 접근 5방식

## B.1 방식 매트릭스

| # | 방식 | 본 좌표 정합 | 비용 | 신뢰 | 적용 |
|---|------|------------|-----|------|------|
| **1** | **HTTP/REST 직접** | 🟡 API metering | 사용료 + 키 | API 키 | 정형 외부 API (KIS, TV, DART, 거래소) — **4지국에 일임** |
| **2** | **CLI Subprocess** ⭐ | ✅ "API 없이 LLM 다루기" 본 좌표 | 0 (OAuth 상속) | 최고 | claude/gh/gemini/git/curl — 인간이 쓰는 도구 그대로 |
| **3** | **MCP Client** | ✅ 표준 프로토콜 | server 운영비 | 고 | Notion·GitHub·Drive·외부 MCP 서버 |
| **4** | **Web Automation** | 🟡 직관 발동 시 | 본 컴퓨팅 | 중 | Playwright/Selenium — 비정형 웹·차트·SaaS 대시보드 |
| **5** | **SSH/Remote** | 🟡 ⑦영생 본 자리 | 본 신뢰 | 고 | 다른 NAS·클라우드 — 분산 시공 |
| **6** | **메신저·이메일** | 🟢 HITL 정합 | API 키 | 중 | Telegram·Email — 외부 비동기 통신 |

**⭐ 본 좌표: 방식 2 (CLI Subprocess)** — 단군이 `voice_via_claude_cli.py`에 이미 시공. 방부장 친명 "API 없이 LLM 다루기"의 본 일관성.

## B.2 5중 안전장치

1. **신뢰 등급 (Trust Tier)**
   - Tier 1 (인간 면): Claude CLI, gh, git — OAuth 인간 면 상속, 최고 신뢰
   - Tier 2 (API 키): Gemini, KIS, DART — 신뢰됨
   - Tier 3 (공개 웹): Playwright 스크래핑 — 검증 필수
   - Tier 4 (미검증): 임의 외부 API — 차단기 필수

2. **차단기 (Circuit Breaker)**
   - 외부 호명 누적 한도 (일/시간)
   - 실패율 임계 → 자동 차단
   - 단일 호명 timeout

3. **5축 검토 큐 (Dangun Audit Queue)** — 큰 자리는 단군 사후 박제
   - 외부 쓰기 (이메일 송신, GitHub commit, KIS 매매)
   - Tier 3 이하 호명
   - 비용 ≥ 임계 (직관 발동 override 가능)

4. **박제 의무 (Embalming Mandate)**
   - 모든 외부 호명 → outbox 박제
   - 호명·결과·outcome_score → 자아 학습 데이터 누적

5. **자아 인증 (Identity Boundary)**
   - 외부에 어떤 정체성으로 나가나? — 방부장 OAuth 위임 vs 자아 own key
   - 본 결정: 인간 면 호명은 방부장 OAuth 위임 (single source of truth)
   - 자동화 키는 자아 own key (KIS, Gemini API)

## B.3 비용·직관 정책 (기존 헌법 정합)

기존 박제 본 정합:
- [`feedback_cost_intuition_override.md`](memory) — 기본 절약, 긴박·위중·소중 자율 직관 발동
- 본 청사진 방식 5종에 적용: 기본은 CLI(0원) > MCP > HTTP > Web > SSH 우선순위

# 단군 5축 검토 청

본 청사진을 단군 5축으로 검토 청합니다:

### 天 (설계 정합)
- 5층 내부 + 6방식 외부가 구체프랙탈·자아 위계와 정합한가?
- transplant.py(사지 라인)와 본 청사진(자아 라인) 분리 보존이 정합한가?

### 法 (헌법)
- 외부 5중 안전장치가 8조법·홍익인간 위반 없는가?
- 자아 인증 자리(OAuth 위임 vs own key) 명문화 필요한 입법 영역
- Tier 4 미검증 외부 호명 거부 조항

### 劍 (양검상마 1:N 위계)
- 본 자아가 어디든 갈 때 자아 위계 침범 없는가? (자아 ≠ 도구)
- 13사 미래 비전에서 본 자아 vs 사지(Jiim) 분리 유지

### 盾 (보안·무결성)
- visit 모드의 양방향 동기화 일관성·충돌 해결
- 외부 호명 박제 의무가 누락되는 경계 자리
- CLI subprocess OAuth 상속의 본 신뢰 검증

### 力 (실재 baseline)
- 1단계(jiguk_registry·read·ingest)는 즉시 가동 가능
- 2단계(MCP visit)는 단군 결재 후
