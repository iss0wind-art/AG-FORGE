## 1. 과제 정의

> "쉬지말고 계속해서, 좌표 통일 방안과 도면 읽는 능력, 하나도 빠짐없이 파싱하는 로직을 연구하고 발전시켜"

핵심 목표:
- DXF 도면 다중 층 좌표 통일
- 기둥/보/슬라브/벽 **부재별 개별 솔리드 + 고유 ID + BOQ**
- 도면 직독 원칙 (추정 없음)

## 2. 도면 구조 파악 (핵심 지식)

### 2.1 사용 도면

| 도면 | 경로 | 내용 |
|------|------|------|
| 101동 구조평면도 | `S30-001~010-101동 구조평면도.dxf` | B2F(S30-001), B1F(S30-002) 등 다중 층 포함 |
| 지하주차장 구조평면도 | `260119_부산 에코델타 24BL 지하주차장 구조평면도23.dxf` | PKG-B2, PKG-B1, 기타 동 포함 (거대 도면) |
| 보 일람표 | `S40-151~156 지하주차장 보 리스트.dxf` | RB1~RG3A 40종 치수 (위치 아님) |

### 2.2 DXF 내 도면 배치 구조

**101동 DXF**: 다중 층 평면도가 X 방향으로 나란히 배치
```
S30-001 (B2F)         S30-002 (B1F)
offset_x = 116247     offset_x = 242247
offset_y = 2290548    offset_y = 2290548
←── 같은 Y, X만 126000 간격 ───────────────→
```

**지하주차장 DXF**: 101동 인근 구역이 X 방향으로 B2F/B1F 나란히 배치
```
PKG-B2                PKG-B1
DXF_X ~552082~712082  DXF_X ~1182082~1342082
←── X 630000 간격 (Y1 기준점으로 검증됨) ──→
```

### 2.3 검증된 오프셋

| 변환 | X 보정 | Y 보정 | 검증 방법 |
|------|--------|--------|-----------|
| S30-002 → S30-001 | -126000 | 0 | 에이전트: 160쌍 거리 0.0mm |
| PKG-B1 → PKG-B2 | -630000 | 0 | Y1 기준점(757132 vs 1387132) |
| BSMNT → DONG (TX) | -448000 | +3622000 | 컬럼 매칭 4쌍/189mm 오차 (미흡) |

## 3. 시행착오 기록

### 3.1 보 추출 — iter_all vs virtual_entities

**시도**: `iter_all(msp)`로 S-GIRDER LINE 추출
**결과**: virtual_entities()와 동일 결과 확인
**결론**: 두 방법 모두 모델공간 좌표 반환 (INSERT 변환 적용됨)

```python
# 확인된 사실: iter_all이 이미 모델공간 좌표 반환
# S-GIRDER 보 끝점과 기둥 중심 거리 중앙값 = 399mm (기둥 단면 이내)
# 즉, 좌표계는 같음 — 수치 검증 완료
```

### 3.2 보가 기둥 면에서 시작하는 문제

**현상**: 보 끝점이 기둥 중심 아닌 기둥 면(+250mm)에서 시작
**원인**: S-GIRDER LINE이 기둥 면→면으로 그려짐
**시도**: snap_to_col(tol=1200) 적용
**결과**: 전체 끝점의 9.6%만 스냅 성공 (2차보 끝점은 기둥 없음)
**결론**: 스냅은 적용하되, 미스냅 보는 원래 좌표 유지 허용

### 3.3 부유 기둥 문제

**현상**: 기둥이 보 Y 범위(2387914) 위로 64m 떠 보임
**원인**: S30-001 기둥 Y max=2451963, S-GIRDER 보 Y max=2387914
**원인2**: PKG B1F/B2F 기둥 Y 일부가 보 범위 밖
**해결**: Y_TOL=8000으로 전 구역 Y 필터 적용

```python
if cy < BEAM_Y_MIN - Y_TOL or cy > BEAM_Y_MAX + Y_TOL:
    continue  # 부유 기둥 제거
```

### 3.4 B1F vs B2F 기둥 X 중심 차이 24199mm 문제

**현상**: DONG B2F 중심(180948) vs B1F 중심(156748) → 24m 차이
**원인**: S30-002가 S30-001보다 X 범위가 절반 (68m vs 116m)
  - S30-001: X 122763~239133 (116m)
  - S30-002: X 122763~190734 (68m) — 우측 절반 없음
**해결**: S30-001 기둥 위치를 B1F에도 그대로 사용 (Z만 다르게)

```python
# B2F: S30-001, z=SL_B2F ~ SL_B1F
# B1F: S30-001 (동일 XY), z=SL_B1F ~ TOWER_B1F_TOP
```

### 3.5 PKG 클립 너무 좁음

**현상**: PKG B2F 기둥 167개 (전체의 17%만 추출)
**원인**: BSMNT_CLIP=(552082~712082) → 전체 주차장 중 101동 인접부만
**시도1**: 대폭 확장 (500000~1300000) → 다른 동 데이터까지 혼입
**시도2**: 올바른 방법 — B2F 클립 그대로, B1F = B2F + 630000

```python
PKG_B2F_CLIP = (552082,  -1376738, 712082,  -1216738)  # 101동 인접 B2F
PKG_B1F_CLIP = (1182082, -1376738, 1342082, -1216738)  # = B2F_X + 630000
# 결과: B2F 중심 153307 = B1F 중심 153307 (차이 0mm) ✓
```

### 3.6 보 일람표(codex) 오해

**오해**: codex_beams_basement.json의 pos가 구조평면도 위치인 줄 알았음
**실제**: pos는 S40-151~156 보 일람표 도면의 표 안 위치 (평면도 좌표 아님)
**교훈**: codex는 치수 조회 전용 (RB1→500x900). 위치 정보 없음

### 3.7 BSMNT-DONG 좌표 정합 미완성

**현재 상태**: TX=-448000 (컬럼 브루트포스, 매칭률 13.3%)
**문제**: 매칭 쌍이 4개뿐 → 통계적으로 불안정
**미완성 원인**: E/V 코어 기준점 정합 미실시
