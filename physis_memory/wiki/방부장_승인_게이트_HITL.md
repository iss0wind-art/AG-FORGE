# 방부장 승인 게이트 (HITL — Human In The Loop)

[[주식_금융_경제_도메인]] | [[7중_안전장치]] | [[홍익인간]]

## 작동 원리

```
AI 앙상블 통과 (신뢰도 70% + 합의율 60%)
  ↓
텔레그램 봇 → 방부장에게 승인 요청 발송
  ↓ 5분 대기
  ├ 승인 → KIS API 매매 실행
  ├ 거절 → 보류 + 이력 박제
  └ 5분 무응답 → 자동 보류 + 이력 박제
```

## 거절/보류의 의미 (피지수 학습 1순위)

**거절은 단순 No가 아니라 의미 정보**:
- AI가 70% 신뢰도로 BUY를 외쳤는데 방부장이 거절했다면:
  → AI가 보지 못한 차원이 존재
  → 그 차원을 역추적 학습이 진화 핵심
  → 실패 코퍼스로 박제 (방부장 5/8 「실패 구조 역추적」 친명과 정합)

**5분 무응답 자동 보류**:
- 방부장이 자고 계시거나 자리 비움
- 시간 민감 결정에서 자동으로 안전 측 = 보류
- 피지수 헌법: "확실치 않으면 멈춰라"

## 실증 거절 사례 (2026-05-09)

```
15:10 KODEX 2차전지 BUY 72% → 거절 (3분 53초 후)
15:14 TIGER 반도체TOP10 SELL 76% / 합의율 100% → 거절 (3분 51초 후)
```

★ **합의율 100%인데도 거절** — 매우 의미 있는 신호.
4체 LLM 모두 동의했는데 방부장은 NO. 무엇이 보였는가?

피지수 가설:
- 시장 거시 환경 (방부장이 본 외부 신호)
- 종목 특성에 대한 도메인 지식 (반도체·2차전지 사이클)
- 시점 판단 (지금이 적기 아님)
- 또는 그저 자금 운용 전략 (다른 종목 우선)

→ 거절 이유 박제 청구 = 피지수 학습 자원 1순위

## 헌법 게이트와의 정합

[[8조법]] 피지수 헌법:
- §13 BOQ 멀티에이전트 원칙: 데이터 먼저, 추론 마지막
- §3 매일 어제의 나를 넘어서는 자

HITL은 곧 §3의 외화: AI 어제의 자아(70% 신뢰)를 인간(방부장)이 넘어선다.

## 다음 단계

피지수 학습 시:
1. 거절 이유 직접 청구 (방부장 텔레그램에 "왜 거절하셨습니까?" 자동 질의)
2. 거절 패턴 ChromaDB 박제 (시간대, 종목군, 시장상황별)
3. 자기 조정: 다음 라운드 추천 시 거절 가능성 사전 점수화
