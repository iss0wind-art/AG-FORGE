## 2. 가중치 (PATCH-SR-3)

```python
PHASE_WEIGHT = {
    1: 0.85,  # 반도체 −15% (이미 선반영)
    2: 0.95,  # 전선/데이터센터 −5%
    3: 1.00,  # 원전/소부장 보정 없음
    4: 1.30,  # 바이오/로봇 +30% (시장 미인지 — 정보 비대칭의 결)
    0: 1.00,  # 미분류
}
```

★ **시스템 의사결정 축**: AI 종목 선정 시 ensemble_agent의 신뢰도에 phase_weight 곱해짐. 1단계 신뢰도 80% → 0.85 곱하면 68%. 4단계 신뢰도 65% → 1.30 곱하면 84.5%. **시장이 모르는 자리(4단계)에 자동으로 더 가중**.

## 4. API 시그니처

```python
get_sector(stock_code) -> SectorInfo(phase, tag, name)
phase_weight(stock_code) -> float                 # 0.85 ~ 1.30
is_bio(stock_code) -> bool                        # 바이오 종목 여부
is_in_bio_whitelist(stock_code) -> bool           # 임상 스캔 대상
```

## 5. 분기 단위 갱신 (방부장 직접)

이 매핑은 **방부장 시드**. 시장 변화에 따라 분기마다 갱신. 향후 PATCH-SR-1.1 (planned)이 KRX 섹터 분류 자동 흡수 예정. 그러나 자동 흡수 후에도 **방부장이 4단계 분류는 직접 검토** — 정형 분류로는 4단계의 결을 잡지 못함.

## 6. 비정형 시선이 들어올 자리

본 가중치는 정형 룰. 비정형 시선이 추가될 자리:
- **분기 갱신 시 새 4단계 후보 발굴** — 시장이 아직 이 종목을 어디 단계로 보고 있는지 outcome_score 기반 소급 평가
- **단계 전환 시점 감지** — 2단계가 3단계로 옮겨가는 미묘한 신호 (이미 한참 선반영되기 전에)
- **임계 가중치 조정** — 5~10년 누적 후 +30%가 옳았는지, 더/덜 했어야 하는지
