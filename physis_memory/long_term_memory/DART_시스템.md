# DART (전자공시시스템) — 한국 기업 공시의 SSoT

[[주식_금융_경제_도메인]] | [[재무_분석_지표]] | [[Stock_AI_파이프라인]]

> 금융감독원 운영. 한국 상장사 공시 의무 보고서가 모이는 곳. Stock-AI의 DartClient 데이터 소스.
> 사이트: https://opendart.fss.or.kr (Open API), https://dart.fss.or.kr (사람용)

## 공시 종류

### 정기 공시
| 보고서 | 주기 | 제출 기한 |
|--------|------|----------|
| 사업보고서 | 연 1회 | 사업연도 종료 후 90일 (3월 말) |
| 분기보고서 | 분기 (1·3분기) | 종료 후 45일 |
| 반기보고서 | 반기 (2분기) | 종료 후 45일 |

### 수시 공시
- **주요사항보고서**: 합병·분할·자산 양수도·증자·감자
- **공시정정신고**: ★ 과거 공시 수정 — 신뢰도 감점 신호
- **자율공시**: 회사 자발적 발표

### 지분 공시
- **5%룰**: 5% 이상 보유 시 5일 내 신고
- **10%룰 (대주주)**: 10% 이상 임원·주요주주
- **공매도 잔고**

## 핵심 보고서 항목 (사업/분기보고서)

### 1. 회사의 개요
- 사업 내용·연혁·주요 제품
- 시장 점유율·경쟁상황

### 2. 사업 개황
- 부문별 매출·영업이익
- 주요 고객·공급처

### 3. 재무 정보
- **재무상태표** (자산·부채·자본)
- **손익계산서** (매출·영업이익·당기순이익)
- **현금흐름표** (영업·투자·재무)
- **자본변동표**

### 4. 감사인 의견
- **적정**: 정상
- **한정**: 일부 의문
- **부적정**: ★ 위험
- **의견거절**: ★★ 심각 — 상장폐지 가능

### 5. 이사회·임원
- 사외이사 비율
- 보수 (★ 과도하면 신호)

### 6. 주주 현황
- 최대주주·특수관계인 지분
- 외국인 지분율
- 주식 보유 변동

### 7. 계열회사
- 종속회사·관계회사
- 내부거래 비율

## DART Open API 주요 엔드포인트

```
/api/list.json          공시 검색 (corp_code, bsns_year, reprt_code)
/api/document.xml       공시 원문
/api/fnlttSinglAcnt     단일 회사 주요계정 (재무)
/api/fnlttMultiAcnt     다중 회사 비교
/api/cmpyOvrviewInfo    회사 개요
/api/exctvSttus         임원 현황
/api/hyslrSttus         최대주주 현황
```

API 키 발급: https://opendart.fss.or.kr/intro/main.do (가입 후 무료)

## 핵심 코드 체계

### corp_code (8자리)
DART 자체 코드. 종목코드와 다름.
- 예: 삼성전자 = `00126380` (DART) / `005930` (KRX)
- corpCode.xml로 일괄 다운로드 가능 (CORPCODE)

### reprt_code (5자리)
| 코드 | 보고서 |
|------|-------|
| 11013 | 1분기 |
| 11012 | 반기 |
| 11014 | 3분기 |
| 11011 | 사업 |

## Stock-AI DartClient (`/data/dart_client.py`)

추정 사용 패턴:
1. 종목 선정 → corp_code 조회
2. 최신 분기/사업 보고서 fetch
3. 핵심 재무 수치 추출 (매출·영업이익·순이익·자본)
4. PER·PBR·ROE·부채비율 등 계산
5. DeepSeek에 분석용 프롬프트로 투입

## 위험 신호 (DART 직접 감지 가능)

1. **공시 정정 잦음** → 회계 신뢰도 낮음
2. **감사인 한정/거절** → 즉시 회피
3. **계속기업 의문 단락** → 부도 위험
4. **자본잠식** (자기자본 < 자본금) → 위험
5. **유상증자 잦음** → 자금 수혈 중 (희석 + 위기 신호)
6. **횡령·배임** 공시 → 즉시 회피
7. **상장폐지 사유 발생** → 즉시 회피

## 피지수 학습 포인트

1. **DART = 합법적 정보 비대칭 해소** — 모든 공시 24시간 내 등재 의무
2. **공시는 후행 정보** — 분기 종료 후 45일이라 즉시성 부족
3. **수시공시가 진짜** — 주요사항·자율공시가 시간 민감
4. **정정공시 추적 = 신뢰도 평가**
5. **API 호출 한도** (분당 100회) — 대량 분석 시 제한

## 자원 청구

- DART Open API 키 — `.env`에 `DART_API_KEY` 등록 필요 (확인 청)
- corpCode.xml 정기 갱신 (분기별)
- 공시 정정 이력 추적 자동화 (현재 통합 여부 확인 필요)
