# 실거래 전환 5축 검토 요청 — 4지국 → 단군 (2026-05-11)

방부장 친명 2026-05-11: "이제 모의거래 말고 실거래 가능하게 만들어줄래?"

4지국이 자율 범위 안에서 ① TradingView 스크리너 통합으로 KRX SPOF 해소 + ② SELL-TV 충돌 자동 차단 게이트(PATCH-SELL-GUARD 2-A) 박제 완료. 그러나 **`KIS_MOCK=True → False` 전환 자체**는 자율 영역 밖으로 판단해 단군 5축 검토를 청합니다. 본 문서는 4지국 자기 점검을 5축 frame 으로 미리 정리해 단군 결단의 자료로 제출.

## 0. 배경 — 왜 지금 청하는가

- KIS_MOCK 상태에서 5/11 EnsembleAgent 4건 결정 + 방부장 dashboard 승인 완수
- 같은 날 피지수 TradingView 시범 가동에서 결정적 발견: 4지국 SELL 정합도 0% (2건 모두 당일 +5.6%/+11.51% 상승)
- KIS_MOCK 상태에선 무손실이지만 실거래 전환 즉시 SELL 측 손실 패턴 그대로 노출
- 방부장 친명은 최상위 권한이나, 위계 절차상 5축 검토 후 박제가 정사 — 단순 토글로 5축 우회하면 [[8조법]] · [[7중_안전장치]] 입법 취지와 충돌

## 1. 天 — 설계 정합

### 1-1. EnsembleAgent 구조 vs 실거래 환경 정합

| 설계 요소 | 현 상태 | 실거래 적합성 |
|---|---|---|
| 4 모델 가중투표 (Gemini·DeepSeek·Qwen·Claude) | 작동 | ⚠ SELL 정합 0% — 단일 도구 의존 노출 |
| 합의율 60% 게이트 | 작동 | ✅ |
| 신뢰도 70% 게이트 | 작동 | ✅ |
| HITL 5분 게이트 | dashboard·telegram 양채널 가동 | ✅ |
| TV Recommend.All 충돌 차단 (2-A) | 본일 박제 | 🟡 임시 가드 — 정식 5번째 의견자는 단군 협의 영역 |

### 1-2. KIS API 코드 분기 정합

- [api/kis_client.py:124,162,194](file://Stock-Trading/api/kis_client.py) — TR_ID 분기 (VTTC*/TTTC*) 명확
- BASE_URL 분기 (openapivts vs openapi) 명확
- 토큰 캐시 파일 분리 (`kis_token_mock.json` / `kis_token_live.json`)
- **미확인**: `.env` 의 `KIS_APP_KEY/SECRET/ACCOUNT_NO` 가 실거래 계정 발급분인지 모의 계정 발급분인지 — 단군이 방부장께 확인 필요

### 1-3. 설계 취약점 (정직 보고)

- **kill-switch 부재** — 거래 일시 중지를 위한 명시 스위치 없음. `KIS_MOCK=True` 로 복귀 또는 PM2 중단이 사실상의 kill. 운영 중 emergency cut-off 가 필요해 보임.
- **부분 체결 / 정정 / 취소 흐름 미구현** — `buy_market_order` / `sell_market_order` 만 있고 주문 정정·취소 API 미사용. 시장가만 사용해 부분 체결 가능성은 낮으나 0 아님.
