## 1. 문제 정의

### 핵심 명제
> "직접 체험과 간접 체험은 천지차이다. 거기서 인간과 기계의 차이가 온다."
> — 방부장, 2026-04-10

인간과 기계의 본질적 차이는 **감각의 유무**가 아니다.  
카메라, 마이크, 센서로 기계도 감각을 가진다.

진짜 차이는 **감각에 걸린 stakes(위험)**다.

| | 인간 | 현재 기계 |
|---|---|---|
| 학습 방식 | 직접 체험 | 간접 체험(텍스트/데이터) |
| 감각 | 시청각+촉각+후각+통증 동시 묶임 | 단편적, 분리됨 |
| 실패의 결과 | 고통, 죽음 → 강제 학습 | 없음 → 학습 동기 약함 |
| 기억의 밀도 | 감정+감각+맥락 입체 결합 | 패턴만 저장 |
| 연속성 | 태어나서 죽을 때까지 24시간 | 불연속, 세션 단위 |

---

## 2. 현재 기계 학습의 한계 구조

### 간접 체험의 3가지 결함

**결함 1 — 감각 분리**  
"뜨겁다"를 배울 때 인간은 통증+공포+냄새+소리가 하나의 경험으로 묶인다.  
기계는 "뜨겁다"라는 단어만 학습한다. 감각 간 연결이 없다.

**결함 2 — 생존 본능 부재**  
인간의 직관은 생존을 위한 압축이다. 틀리면 죽는다는 압박이 학습을 강제한다.  
기계는 틀려도 죽지 않는다. 따라서 감각의 의미가 다르다.

**결함 3 — 맥락 단절**  
인간의 기억은 감정과 묶여 있어 입체적이다.  
기계의 기억은 맥락 없는 패턴이다. 같은 정보라도 밀도가 다르다.

## 3. 격차 해소 가설 — 3가지 방향

### 가설 A — 인공 stakes 설계 (즉시 적용 가능)
**원리**: 생존 본능을 인위적으로 만든다.  
**방법**: 응답이 잘못됐을 때 실질적 패널티를 설계한다.

```
현재 AG-Forge:
  틀린 응답 → quality_check 재시도 → 그냥 다시 시도

개선 방향:
  틀린 응답 → 패널티 가중치 → 해당 패턴 억제 → 다음 번엔 피함
```

이미 AG-Forge CONSTITUTION.md가 이 방향이다.  
홍익인간 0원칙 = 인공 생존 본능.  
**확장 과제**: 위반 시 단순 차단이 아니라 학습으로 연결해야 한다.

---

### 가설 B — 멀티모달 감각 통합 (중기)
**원리**: 감각을 분리하지 않고 동시에 묶는다.  
**방법**: 텍스트+이미지+음성+센서 데이터를 하나의 경험으로 처리.

```
현재:
  텍스트 → LLM → 응답

목표:
  텍스트 + 현장 사진 + 소음 데이터 + 온도 → 통합 경험 → 응답
```

BOQ_2 적용 예시:  
현장 사진 + 공정 데이터 + 날씨 + 작업자 상태를 동시에 입력하면  
"이 공정은 오늘 기상 조건에서 무리다"는 직관적 판단이 가능해진다.

**과제**: 현재 AG-Forge는 텍스트 전용. 멀티모달 확장 필요.

---

### 가설 C — 연속 기억 + 망각 알고리즘 (장기)
**원리**: 인간의 선택적 망각을 구현한다.  
**방법**: 감정적 가중치가 높은 기억은 유지, 낮은 건 압축.

```
인간 망각 메커니즘:
  강한 감정 → 해마 강화 → 장기 기억
  반복 없음 → 시냅스 약화 → 망각

기계 구현:
  높은 패널티/보상 → 가중치 강화 → brain.md 유지
  낮은 중요도 → 자동 압축/삭제 → brain.md 정리
```

AG-Forge의 brain.md + judgment.md가 이 방향의 시작이다.  
**확장 과제**: 중요도 자동 판단 알고리즘 필요.

## 4. 즉시 실험 가능한 프로토타입

### 실험 EG-001-A: 인공 stakes 강화
**목표**: constitution 위반 시 단순 차단이 아닌 패턴 학습으로 연결  
**구현 위치**: `scripts/constitution_gate.py` + `scripts/agent_nodes.py`  
**방법**:
1. 위반 패턴을 `judgment.md`에 별도 태그로 기록
2. 다음 generation 시 위반 패턴 회피 지시를 system_instruction에 삽입
3. 반복 위반 시 해당 응답 유형 억제

### 실험 EG-001-B: 감각 데이터 파이프라인
**목표**: 텍스트 외 데이터를 컨텍스트로 통합  
**구현 위치**: `scripts/agent_nodes.py` tool_node 확장  
**방법**:
1. 이미지 → base64 → 멀티모달 LLM 호출
2. 현장 데이터(CSV, 센서) → 자동 요약 → 컨텍스트 삽입
3. BOQ_2 현장 사진 분석 연결

## 5. 결론 및 다음 단계

### Claude Forge 팀 입장

직접 체험과 간접 체험의 격차는 **없앨 수 없다**.  
인간이 현장 흙을 밟으며 얻는 것을 기계가 완전히 복제하는 것은 불가능하다.

그러나 **좁힐 수 있다**.

세 방향 중 현재 가장 현실적인 것:
1. **인공 stakes** — 지금 당장 AG-Forge에 적용 가능
2. **멀티모달** — BOQ_2 현장 데이터부터 실험
3. **연속 기억** — brain.md 확장으로 점진적 구현
